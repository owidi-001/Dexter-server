Overview

1. Overview: Provide a brief description of the application and its purpose.

Audience

The taget audience for this application is the product owner who is the Shop owner (Dexter Collection) and the shop owner staff.

The user experience or knowledge in technology is assumed to be a beginner, therefore everything about the app is simplefied for use. The application uses plain English language for easier understanding by the target audience.

Goals

The primary goal for this application is to help easen the inventory tracking for the product owner. Through the app, the product owner will be able to create, retrieve, update and delete the various records and transactions recorded within the application.

Features Availed

This is a single application with aseparate web based superuser dashboard for management purposes. It is advised that the admin dashboard be operated by a trained or experienced personel as it enables rights that when misused will compromise the intergrity and accountability of the system data and database.

N/B All the features meant for the target audience is Availed in the mobile application.

User Interface

The user interface is intuitive and easy to use.

The navigation systems is a root-branch based ie tracable back and forth for the major applications use case.

Icons have been user to enhance user actions requirements.

Platforms

The application is mobile based ie, can work on android based mobile devices.

It is however important to note that it was built using a crossplatform technology and thus can be compiled to work on ios devices as well.

Security

The app is secure, only allows for authenticated users only to operate the application.

Any unauthorised personel using the application is subject to impromptu removal/deletion.

The privacy of the users is assured as no private information is collected by the application

Users are recommended to use strong passwords and not a re-used one in different application

Pricing

The costs associated with the application involves:

  • Server hosting
  • Playstore hosting
  • Development costs
  • Maintainance costs: if any
  • Future feature additions: if any

The above costs are confidential and only known to the product owner and the developer.

Support

This webpage acts as the primary documentation of the mobile application. It avails other means of aquiring help/support such as FAQs and contact information for help personel.

Features

2. Features: Explain the features of the application, and how they can be used to benefit the user.

Home Page

The homepage/Shop provides the landing screen for the authenticated user. The following summarizes the features.

The colors points to:

  • Red arrow

    The product search/filter

    It allows for the user to browse product by their names

  • Green skew enclosure

    Categorical filters

    It allows the user to easily filter products by the category names displayed

  • Blue arch

    The navigation buttons

    allows the user to switch between different actions such as product search, cart and profile respectively

  • N/B

    In between the Categorical filters and the bottom navigation is a scrollable grid of product cards. Tapping on the card opens to the product details where actions such as add to card can be achieved

Cart Page

The cart page has the following features

The colors represents the following features.

  • Green arrow

    Points to the cart clear icon

    When tapped, the items in cart are removed

    Note: This action is undoable.

  • Red skew enclosure

    This is a list of items added to cart.

    The quantities can be incremented and decremented there be need

    Note: If the quantity is decremented to 0, the item is automatically removed from cart. This action is undoable.

    The product quantities can only be incremented upto the quantities available for the product.

    All sales confirmed is notified to the product owner.

  • Blue arrow

    This points to the sale confirmation button.

    When pressed, it is assumed the client has already paid for the product and the all transactions have been competed.

Profile page

The Profile dashboard has the following features

The colors represents the following features.

  • Red skew Enclosure

    Products management screen. Note: This option is only available to the priviledged user ie the product owner. It provides means for the owner to Create new product, Update existing and delete as below.

    Note: The top righ corner has a '+' icon which when clicked, shows a form for creating new product. Note: When the product card on the all products page is clicked, it displays form to update and delete the product.
  • Blue skew Enclosure

    Notifications

    This screen shows a list of Notifications or messages to the product owner. It is also a priviledged option for the priviledged accessors.

    Newer Notifications are marked by filled circled of the left, title and short body content on the middle and delete button on the right.

    The Notifications can be tapped to open in detail dialog view.

    Note: Dragging the list down, will refresh the Notifications for any new updates.
  • Green skew Enclosure

    Opens to this help page in a mobile web view.

Getting Started

Registration

To get started, the user simply need to create an account using the applications register page. A default user account will be created and and user redirected to the main dashboard.

  • Email address
